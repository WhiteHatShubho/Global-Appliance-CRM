STEP 3: Update Firebase database.rules.json
================================================================================

Add this rule to prevent audit log deletion:

```json
{
  "rules": {
    "auditLogs": {
      ".read": "auth != null",
      ".write": "auth != null",
      "$logId": {
        ".delete": false
      }
    }
  }
}
```

================================================================================
                         TROUBLESHOOTING
================================================================================

Issue: "Audit Service not initialized" warning
Solution:
- Make sure Login.js has auditService.initialize() call
- Check browser console for errors during login

Issue: Audit logs not showing
Solution:
- Make sure database is accessible
- Check Firebase rules allow reads on auditLogs
- Verify user is logged in with valid auth token

Issue: Recovery showing empty
Solution:
- No items deleted yet (this is good!)
- Try deleting a test customer
- Refresh the recovery page

Issue: Backups not working
Solution:
- Check browser storage quota
- Browser may have limited localStorage
- Try opening DevTools > Storage > IndexedDB
- Verify backupService.initialize() was called
